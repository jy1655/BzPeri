Install from the APT repo

Use the signed repository for Debian or Ubuntu systems, then enable BlueZ experimental support if your host still needs it. 

curl -fsSL https://jy1655.github.io/BzPeri/repo/repo.key | \
  sudo gpg --dearmor -o /usr/share/keyrings/bzperi-archive-keyring.gpg

echo "deb [signed-by=/usr/share/keyrings/bzperi-archive-keyring.gpg] \
https://jy1655.github.io/BzPeri/repo stable main" | \
  sudo tee /etc/apt/sources.list.d/bzperi.list

sudo apt update
sudo apt install bzperi bzperi-dev bzperi-tools
sudo /usr/share/bzperi/configure-bluez-experimental.sh enable

Manual run-loop control bzpStartManual(), owner handoff helpers, poll-cycle APIs, and bounded drive helpers for hosts that need tighter lifecycle control.

Quick start after install

sudo bzp-standalone -d

# Inspect adapters on the host
sudo bzp-standalone --list-adapters

# Build from source with tests
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release -DBUILD_TESTING=ON
cmake --build build --parallel
ctest --test-dir build --output-on-failure
